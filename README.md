# Multilayer Perceptron (MLP) from Scratch

A neural network implemented entirely from scratch using only NumPy (no TensorFlow/PyTorch). Built to classify breast tumors as **Malignant (M)** or **Benign (B)** using the Wisconsin Breast Cancer Dataset (569 samples, 30 features).

## How It Works

1. **Data split** ([split.py](split.py)): Shuffles `data.csv` with a fixed seed and writes `train.csv` (80%) and `validation.csv` (20%).
2. **Data loading** ([main.py](main.py)): Reads the CSVs, extracts 30 numeric features (columns 2+), and one-hot encodes the label (`M` → `[0,1]`, `B` → `[1,0]`).
3. **Standardization** ([model.py](model.py)): Z-score normalization is applied inside `Model.fit()`. **Training mean/std are stored on the model and reused at inference time** to prevent data leakage.
4. **Network construction** ([network.py](network.py)): Builds a fully-connected network with configurable hidden layers (default: 3 layers of 24 neurons). Weights are initialized with **He Uniform** (default) or **Xavier Uniform**.
5. **Forward pass**: Hidden layers use a configurable activation (ReLU or sigmoid); the output layer uses **softmax**.
6. **Backward pass**: For cross-entropy + softmax, the gradient simplifies to `(y_pred - y_true) / batch_size`.
7. **Optimization** ([optimizer.py](optimizer.py)): **SGD** (vanilla gradient descent) or **Adam** (adaptive moment estimation with bias correction).
8. **Training** ([model.py](model.py)): Mini-batch training with shuffling, tracks loss/accuracy for train and val, and **early stopping** (restores best weights after patience runs out).
9. **Output**: Saves `model.pkl` (pickle) and a `learning_curve.png` plot.

## SGD vs Adam — Concrete Numbers

Both optimizers minimize the same loss, but Adam keeps **per-parameter running averages** of the gradient (1st moment, momentum) and its square (2nd moment, RMSProp), with bias correction. SGD applies a single global learning rate to every parameter.

### Update rules

| Optimizer | Update |
|---|---|
| **SGD** | `W ← W − lr · ∇W` |
| **Adam** | `m ← β₁m + (1−β₁)∇W`, `v ← β₂v + (1−β₂)(∇W)²`, `W ← W − lr · m̂ / (√v̂ + ε)` |

### Benchmark 1 — Synthetic overfit (50 samples, 300 epochs, full-batch)

Reproduces [notebook/03_adam_vs_sgd.ipynb](notebook/03_adam_vs_sgd.ipynb).

| Optimizer | Learning rate | Final loss | Epochs to loss < 0.5 | Epochs to loss < 0.1 | Final accuracy |
|---|---:|---:|---:|---:|---:|
| **Adam** | 0.001 | **0.001991** | **20** | **87** | **100.0 %** |
| SGD | 0.01 | 0.284359 | 109 | never | 98.0 % |
| SGD | 0.001 | 0.648458 | never | never | 62.0 % |

- **Convergence speed**: Adam reaches `loss < 0.5` in **20 epochs** vs SGD's **109 epochs** — **~5.5× faster**.
- **Final loss**: Adam's `0.00199` is **~143× lower** than the best SGD result (`0.284`).
- **LR sensitivity**: dropping SGD's lr 10× (0.01 → 0.001) collapses accuracy from 98 % to 62 %. Adam is far more forgiving — the same `lr=0.001` that breaks SGD is Adam's sweet spot.

### Benchmark 2 — Real data (Wisconsin Breast Cancer, 200 epochs, batch=4, seed=42)

| Optimizer | Learning rate | Val loss | Val accuracy |
|---|---:|---:|---:|
| **Adam** | 0.0001 | 0.1882 | **96.49 %** |
| SGD | 0.0001 | 0.1512 | 94.74 % |
| SGD | 0.001 | **0.0994** | 95.61 % |

On this small, well-conditioned dataset SGD with a well-tuned lr is competitive on loss, but **Adam wins on accuracy without any LR tuning** — that is the practical reason it is the default `--solver`.

### Takeaway

- **Default to Adam** when you cannot afford a learning-rate sweep — it adapts the effective step per parameter from the gradient history.
- **SGD requires careful LR tuning** (often a schedule). With the right lr it can match Adam, but the cost of getting there is real.
- Adam's momentum (`m`) accelerates descent in consistent directions; its `v` term shrinks the step where gradients are noisy. SGD has neither.

## Improvements Over the Initial Prototype

This codebase went through several correctness and reproducibility fixes — see `git log` for the full history. Highlights:

| Fix | Why it matters |
|---|---|
| **Fixed Adam optimizer update** (`fix: adam optimizer weights and biases update`) | The original update reused the running averages incorrectly. After the fix, Adam converges to `loss < 0.002` on the overfit benchmark instead of plateauing. |
| **Eliminated data leakage** (`fix: data leakage, double standardization`) | Mean/std are now computed **only on train**, stored on the model, and applied to validation/test. Inference no longer re-normalizes with test-set statistics — predictions are stable and honest. |
| **Fixed network weight init & batched gradients** (`fix: network weights init and gradient batches`) | Gradients are now divided by `batch_size`, so the effective learning rate doesn't drift with batch size. He Uniform / Xavier Uniform are picked correctly per init flag. |
| **Added `--seed` argument** | Training is now bit-for-bit reproducible across runs. |
| **Added `split.py`** | Deterministic 80/20 split with a configurable seed — no more silent shuffles. |
| **Migrated sanity checks to a pytest suite** (`refactor: migrate validation notebooks to pytest test suite`) | Activation, gradient, optimizer, IO, and stability checks now run as `tests/test_*.py` — see `run_tests.sh`. |

## Design Decisions

**Why softmax + cross-entropy only?** The combined derivative simplifies to `(y_pred − y_true)`, which is numerically stable and cheap. Softmax + MSE pulls in the full softmax Jacobian and risks vanishing gradients. Unsupported combinations raise an error — fail-fast over false flexibility.

**Why ReLU as the default hidden activation?** Modern default for deep networks. On small datasets sigmoid can be competitive (measured: sigmoid val_acc ~97.4 % vs ReLU ~95 %). Both are supported via `--activation`.

**Why does `backward` receive `z` (pre-activation)?** The chain rule defines the activation derivative in terms of `z`. Sigmoid's derivative recomputes sigmoid internally, but the cost is negligible. Accepting `z` keeps the interface agnostic to activation type.

**Why separate `Model` and `Network`?** `Model` owns the training loop, standardization, and serialization. `Network` owns forward and backward only. Splitting responsibilities keeps each class focused and independently testable.

## Default Configuration

- Hidden layers: `[24, 24, 24]`
- Hidden activation: ReLU
- Weight initialization: He Uniform
- Output: 2 neurons (softmax)
- Loss: Cross-entropy (only supported loss)
- Optimizer: Adam (lr = 0.0001)
- Batch size: 4 — Epochs: 200
- Early stopping patience: 10 epochs
- Random seed: 42

## Usage

### 1. Split the dataset

```bash
python split.py data.csv --ratio 0.8 --seed 42
```

Produces `train.csv` and `validation.csv`.

### 2. Train

```bash
python main.py --data train.csv --val_data validation.csv \
  --layers 24 24 24 --epochs 200 --batch_size 4 \
  --learning_rate 0.0001 --solver adam --activation relu \
  --weights_initializer heUniform --seed 42
```

| Argument | Description | Default |
|---|---|---|
| `--data` | Training CSV | `train.csv` |
| `--val_data` | Validation CSV | `validation.csv` |
| `--layers` | Hidden layer sizes | `24 24 24` |
| `--epochs` | Training epochs | `200` |
| `--batch_size` | Batch size | `4` |
| `--learning_rate` | Learning rate | `0.0001` |
| `--solver` | `adam` or `sgd` | `adam` |
| `--activation` | `relu` or `sigmoid` | `relu` |
| `--weights_initializer` | `heUniform` or `xavierUniform` | `heUniform` |
| `--seed` | Random seed | `42` |

Outputs: `model.pkl` and `learning_curve.png`.

### 3. Predict

```bash
python predict.py --data validation.csv --model model.pkl
```

| Argument | Description | Default |
|---|---|---|
| `--data` | Input CSV | `validation.csv` |
| `--model` | Trained model file | `model.pkl` |

### 4. Run the test suite

```bash
./run_tests.sh
```

Covers activation derivatives, gradient correctness, optimizer behavior, IO, numerical stability, and end-to-end training.
